HydroAgent as trusted operational software for flood forecasting.

The manuscript framing matters because it explains the system as a workflow that separates expert judgment, hydrological computation, and human approval.

See results section Open EGU summary
Core claim

The system is judged by workflow performance, not model theater.

HydroAgent is not presented as a flood-predicting chatbot. It is presented as a structured operational layer around models, evidence, and release control.

Judgment

Prior judgment is explicit.

Hydrological expertise is formalized before detailed forecast refinement begins.

Execution

Hydrological computation stays central.

Deterministic models remain part of the core system rather than being replaced by language-model improvisation.

Control

Human approval remains explicit.

The release boundary does not disappear behind automation, which is central to HydroAgent's trust posture.

Current benchmark

Early evidence already supports the workflow thesis.

In the current 14-event benchmark, the workflow completed all cases, preserved the review pause in every case, and improved runoff-volume error in 9 events relative to the Step 0 baseline.
